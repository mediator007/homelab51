# Ansible Role: Bind9 DNS Server

## Цель роли

Эта роль разворачивает DNS-сервер BIND9 в Docker-контейнере. Цель — предоставить внутреннюю DNS-зону для вашей домашней лаборатории, а также подготовить инфраструктуру для вторичного DNS.

## Конфигурационные файлы

1. **named.conf.options** – основной конфиг, где настраиваются общие параметры BIND9: разрешенные сети, включение рекурсивного режима, forwarders (если нужны внешние запросы).
2. **named.conf.local** – здесь определяются локальные зоны, например `homelab.local`, и их тип (master/slave).
3. **Зоны (*.db)** – содержат записи DNS: A, AAAA, CNAME для устройств и сервисов. В master-зоне добавляются все записи, которые должны быть авторитетными.

## Зачем нужны эти конфиги

* `named.conf.options` управляет глобальным поведением DNS-сервера.
* `named.conf.local` говорит серверу, за какие зоны он отвечает и где искать вторичные копии.
* Файлы зон содержат сами DNS-записи, без них сервер не сможет разрешать имена.

## Результат

После выполнения роли вы получите:

* Master DNS-сервер с зоной `homelab.local`;
* Вторичный сервер готовый принимать репликацию;
* Возможность добавлять новые записи через Ansible и шаблоны зон.
